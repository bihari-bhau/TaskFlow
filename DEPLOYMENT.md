# TaskFlow Deployment Guide

## Railway Deployment Setup

### Current Service URLs
- **Frontend**: `https://vibrant-inspiration-production-f24a.up.railway.app`
- **Backend**: `https://taskflow-production-8281.up.railway.app`

### Frontend Environment Variables

To enable API communication between frontend and backend on Railway, configure these environment variables in the **Frontend Service**:

#### 1. `BACKEND_API_URL` (Required)
- **Value**: `https://taskflow-production-8281.up.railway.app`
- **Purpose**: Nginx proxy endpoint for API requests from the browser
- **How it works**: The nginx reverse proxy in the frontend container routes `/api/*` requests to this backend URL

#### 2. `REACT_APP_API_URL` (Optional fallback)
- **Value**: `http://localhost:8000/api` (for local testing)
- **Purpose**: Fallback for browser-based API calls if proxy fails
- **Used by**: The React app's `api.ts` file (via `window.__REACT_APP_API_URL`)

### Backend Environment Variables

Configure these in the **Backend Service**:

| Variable | Value | Purpose |
|----------|-------|---------|
| `DATABASE_URL` | Your Railway PostgreSQL connection string | Database connection |
| `SECRET_KEY` | Auto-generated by Railway | JWT signing key |

### Step-by-Step Setup

1. **Go to your Railway Dashboard**
   - Navigate to your `taskflow-frontend` service
   - Click on the **Variables** tab

2. **Add `BACKEND_API_URL`**
   - Key: `BACKEND_API_URL`
   - Value: `https://taskflow-production-8281.up.railway.app`
   - Click **Add Variable**

3. **Verify Backend URL**
   - Go to your `taskflow-backend` service
   - Under **Settings** → **Service Domain**, note the public URL
   - Update `BACKEND_API_URL` in frontend if it differs

4. **Redeploy Frontend**
   - Click the **Deploy** button in Railway
   - Wait for the build and deployment to complete

5. **Test the Connection**
   - Go to `https://vibrant-inspiration-production-f24a.up.railway.app/login`
   - Try logging in or signing up
   - Check the browser console for errors (F12)

### Troubleshooting

#### "405 Not Allowed" Error
- **Cause**: Nginx proxy not configured or backend URL incorrect
- **Fix**: Verify `BACKEND_API_URL` environment variable is set correctly in Railway

#### "Host not found" Error
- **Cause**: Nginx trying to resolve hostname at build time
- **Fix**: Use environment variables instead of hardcoded hostnames in nginx.conf

#### CORS Issues
- **Cause**: Backend not accepting requests from frontend domain
- **Status**: ✅ Already configured in `backend/main.py` with `allow_origins=["*"]`

#### Blank Page After Login
- **Cause**: Frontend can't reach backend API
- **Debug**: 
  1. Open browser DevTools (F12)
  2. Go to **Network** tab
  3. Try login, check for failed API requests
  4. Look for the actual error in the response

### Local Development

For local development with Docker Compose:

```bash
docker-compose up --build
```

- Frontend runs on: `http://localhost:3000`
- Backend runs on: `http://localhost:8000`
- Nginx automatically proxies `/api/*` to the backend on the Docker network

### Additional Notes

- The nginx proxy runs inside the frontend container and uses the `BACKEND_API_URL` environment variable
- The frontend React app has a fallback `REACT_APP_API_URL` that can be used for direct browser requests (though proxy is preferred)
- Changes to environment variables require a service redeploy
